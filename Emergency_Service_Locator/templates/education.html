Divide & Conquer: Closest Pair of Points

Understanding the core algorithm powering our Emergency Service Locator.

The Problem Statement

Given a set of \(n\) points in a 2D plane, find the pair of points with the smallest Euclidean distance between them. In our context, we are finding the minimum distance between an Emergency Request point and the set of available Service Units (Hospitals, Police, Fire Stations).

Euclidean Distance Formula: d(p1, p2) = √((x2 - x1)² + (y2 - y1)²)

The Naive Approach (Brute Force)

The simplest way to find the closest pair is to calculate the distance between every possible pair of points and keep track of the minimum.

  • Compare point 1 with points 2, 3...n
  • Compare point 2 with points 3...n
  • Requires \(n(n-1)/2\) comparisons.
Time Complexity: O(n²)

The Divide & Conquer Approach

To optimize the search, we can use the Divide and Conquer paradigm. The algorithm works as follows:

  1. Sort: Sort all points by their x-coordinates.
  2. Divide: Split the points into two equal halves by a vertical line.
  3. Conquer: Recursively find the closest pair in the left half and the right half. Let the minimum of these two distances be \(d\).
  4. Combine (The Strip): The closest pair might span across the dividing line. Create a strip of width \(2d\) centered at the dividing line. Sort the points in this strip by their y-coordinates.
  5. Check Strip: For each point in the strip, check the distance to the next 7 points. This mathematical bound ensures we only do a constant number of comparisons per point in the strip.
Time Complexity: O(n log n)
T(n) = 2T(n/2) + O(n)
